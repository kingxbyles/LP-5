Before writing any logic, we must import the libraries (pre-written code) that do the heavy lifting for us.

2. Loading the Data
Python
boston = pd.read_csv("boston_house_prices.csv")

•	pd.read_csv(...): This Pandas function opens your CSV file, reads all the rows and columns, and stores it in the computer's memory as a "DataFrame" (a virtual table) named boston.

3. Feature Selection & Splitting
Python
X = boston[['LSTAT', 'RM', 'PTRATIO']]
y = boston['PRICE']

•	X (Features): We tell the computer, "Look at the table and only extract these three columns to use as clues."
•	y (Target): We extract the PRICE column. This is the "answer key" the model is trying to guess.
Python
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, random_state=4)

•	test_size=0.2: Tells the function to keep 80% of the data for training the model, and hide 20% of the data to test it later.
•	random_state=4: Shuffles the data randomly. Setting it to 4 (or any specific number) acts as a "seed." It ensures that every time you run the code, it shuffles the data in the exact same way, so your results don't randomly change.
________________________________________
4. Data Preprocessing (Scaling)

Python
scaler = StandardScaler() 
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

•	StandardScaler(): Neural networks hate it when one feature has small numbers (like 0.1) and another has huge numbers (like 1,000). This tool mathematically squeezes all numbers so their average is 0 and they vary mostly between -1 and 1.
•	fit_transform(...): Used ONLY on the training data. It does two things: fit calculates the mean and standard deviation of the training data, and transform actually applies the math to shrink the numbers.
•	transform(...): Used on the test data. Crucial Viva Point: Notice there is no fit here! We must use the exact same mathematical rules from the training data to shrink the test data to prevent "Data Leakage" (cheating by letting the model peek at the test data's average).
________________________________________
5. The Deep Neural Network (DNN) Architecture

Python
model = Sequential([
    Dense(128, activation='relu', input_dim=3),  
    Dense(64, activation='relu'),  
    Dense(32, activation='relu'),  
    Dense(16, activation='relu'),  
    Dense(1, activation='linear')  
])
•	Sequential([...]): Creates an empty neural network pipeline where data flows straight through.
•	Dense(128...): Adds a hidden layer with 128 "neurons" (mathematical nodes).
•	activation='relu': ReLU (Rectified Linear Unit) is an activation function. It simply says: "If the output of a neuron is a negative number, turn it into 0. If it is positive, leave it alone." This allows the network to learn complex, non-linear patterns instead of just straight lines.
•	input_dim=3: We must put this on the very first layer. It tells the neural network to expect exactly 3 input features (LSTAT, RM, PTRATIO).
•	Dense(1, activation='linear'): The final output layer. It only has 1 neuron because we only want 1 answer (the house price). We use linear activation because this is a Regression problem. We want the neuron to output the raw, unedited number (like $75,000) instead of a percentage between 0 and 1.
________________________________________
6. Compiling the Model

Python
model.compile(optimizer='adam', loss='mse', metrics=['mae'])  
•	compile: Translates our layer structure into a language the computer's processor can actually calculate.
•	optimizer='adam': Adam is the "teacher" of the neural network. As the network guesses prices and makes errors, Adam calculates how to adjust the weights of the neurons to do better next time. It automatically adapts the "learning rate" to be fast at the beginning and careful at the end.
•	loss='mse': Mean Squared Error. This is how the model calculates its mistakes during training. It subtracts the predicted price from the real price, squares the difference (to remove negative signs and punish huge errors), and averages them.
•	metrics=['mae']: Mean Absolute Error. We ask Keras to also print this out for us because it is easier for humans to understand (e.g., "You are off by $3,000 on average").
________________________________________

7. Training the Model
Python
history = model.fit(X_train_scaled, y_train, epochs=100, validation_split=0.05, verbose=1)
•	fit: The command that actually starts the training loop.
•	epochs=100: The neural network will look at the entire training dataset exactly 100 times, learning a little bit more each time.
•	validation_split=0.05: Tells the model to hold back 5% of the training data as a "mini quiz" at the end of every epoch. This helps us ensure the model isn't just memorizing the answers (Overfitting).
•	verbose=1: Tells Keras to print the live progress bar to the screen. Setting it to 0 hides the text.

Python

Python
new_data = np.array([[0.1, 10.0, 5.0]])  
new_data_scaled = scaler.transform(new_data)

•	np.array([[...]]): Creates a brand new, fake house with specific stats to test the model. The double brackets [[ ]] are required because the model expects a 2D table of inputs, not a 1D list.
•	scaler.transform: We must apply the exact same shrinkage to our fake house stats before feeding it to the network, otherwise the network will be confused by the un-scaled numbers.
•	predict: Asks the trained network to output its final guess for the price.

Part 1: The Basics of Machine Learning & Regression



1. What is Machine Learning?
It is a subset of Artificial Intelligence where the computer learns patterns from historical data without being explicitly programmed with if/else rules.

2. What is the difference between Regression and Classification?
•	Regression: Predicts a continuous numerical value (e.g., predicting House Price, Temperature, Stock Market price).
•	Classification: Predicts a distinct category or class (e.g., predicting if an email is Spam or Not Spam, Dog or Cat).

3. Why is Boston Housing considered a Regression problem?
Because the target variable we are trying to predict (PRICE) is a continuous number (e.g., $24,000, $35,500), not a category.

4. What are Independent and Dependent Variables in this dataset?
•	Independent Variables (Features / $X$): The clues used to make the guess (e.g., LSTAT, RM, PTRATIO).
•	Dependent Variable (Target / $y$): The outcome being predicted (PRICE).

5. What do the features LSTAT, RM, and PTRATIO represent?
•	LSTAT: Percentage of the lower-status population in the area.
•	RM: Average number of rooms per house.
•	PTRATIO: Pupil-to-Teacher ratio in the local schools.

6. What is Traditional Linear Regression?
A statistical method that attempts to draw a straight "line of best fit" through the data points to predict the relationship between $X$ and $y$. Equation: $y = mx + c$.

7. What is Mean Squared Error (MSE)?
It measures the average of the squares of the errors. (Error = Actual Price - Predicted Price). We square it to remove negative signs and to heavily punish large mistakes.

8. What is Mean Absolute Error (MAE)?
It measures the average absolute difference between the actual and predicted prices. If MAE is 3.5, it means the model is off by $3,500 on average.

9. What is the R2 Score ($R^2$)?
The Coefficient of Determination. It grades the model from 0 to 1. An $R^2$ of 0.85 means the model accurately explains 85% of the variance in the house prices.

Part 2: Deep Neural Network (DNN) Concepts


10. What is an Artificial Neural Network (ANN / DNN)?
A computing system inspired by the human brain, consisting of interconnected nodes (neurons) arranged in layers (Input, Hidden, and Output layers).

11. What is a Neuron (or Node)?
A mathematical function that takes inputs, multiplies them by weights, adds a bias, passes the result through an activation function, and sends it to the next layer.

12. What are Weights and Biases?
•	Weights: Determine how "important" an input is. (e.g., Number of rooms has a high weight for house prices).
•	Bias: A constant number added to shift the result, similar to the $c$ (y-intercept) in $y = mx + c$.

13. What is an Activation Function?
A mathematical "gate" at the end of a neuron that decides whether the neuron should activate and pass its signal forward. It introduces non-linearity.

14. Why did we use ReLU in the hidden layers?
ReLU (Rectified Linear Unit) turns all negative values to zero and keeps positive values unchanged. It is computationally fast and prevents the "vanishing gradient problem" during training.

15. Why did we use linear activation in the final output layer?
Because this is a Regression problem. We want the network to output the raw, unedited predicted number (e.g., $45,000). If we used sigmoid, the network would compress the price between 0 and 1, completely ruining the prediction.

16. What is Forward Propagation?
The process of data traveling from the input layer, through the hidden layers, to the output layer to make a prediction.

17. What is Backpropagation?
The core learning mechanism. After making a prediction, the network calculates its error (Loss). It then travels backwards through the layers, adjusting the weights and biases to reduce the error for the next time.

18. What is an Epoch?
One complete pass of the entire training dataset through the neural network (forward and backward).

19. What is an Optimizer?
The algorithm that decides how much to change the weights during backpropagation.

20. Why did we use the Adam optimizer?
Adam (Adaptive Moment Estimation) is the most popular optimizer because it automatically adjusts the learning rate as it trains, making it faster and more reliable than standard Gradient Descent.

Part 3: Data Preprocessing & Scaling



21. Why do we split data into Training and Testing sets?
To test if the model actually learned the concepts, or if it just blindly memorized the answers. The Test set acts as an unseen "final exam."

22. Why did we use random_state=4?
It acts as a random seed. It guarantees that the data is shuffled exactly the same way every time you run the script, ensuring your results are reproducible.

23. What does StandardScaler do mathematically?
It standardizes features by subtracting the mean and dividing by the standard deviation. This forces the data to have a mean of 0 and a standard deviation of 1.

24. Why is Feature Scaling mandatory for Neural Networks?
If RM is 5 (small number) but PRICE is 50,000 (huge number), the neural network's weights will fluctuate wildly and it will take forever to train. Scaling puts all numbers on a level playing field, allowing the Adam optimizer to find the answer quickly.

25. Why do we use fit_transform on train data, but only transform on test data?
fit calculates the average of the data. If we fit the test data, we are letting the model "peek" at the test set's average, which is cheating. This is known as Data Leakage. We must scale the test data using the training data's rules.

Part 4: Keras Code & Programming Specifics


26. What does Sequential() do?
It tells Keras that we are building a linear stack of layers, where layer 1 feeds exactly into layer 2, and so on.

27. What is a Dense layer?
A layer where every single neuron is connected to every single neuron in the previous layer.

28. Why did we specify input_dim=3?
Because we are feeding the network exactly 3 features (LSTAT, RM, PTRATIO).

29. Why does the final layer only have Dense(1)?
Because we only want the network to output a single number: the predicted House Price.

30. What does model.compile() do?
It configures the model for training by linking the Optimizer (Adam) and the Loss Function (MSE) to the network.

31. What is Overfitting?
When a model memorizes the training data perfectly but performs horribly on new test data (like a student memorizing a practice test but failing the real exam).

32. How does validation_split=0.05 help prevent Overfitting?
It holds back 5% of the training data. At the end of every epoch, the model tests itself on this 5%. If the training loss goes down, but the validation loss goes up, you know the model is starting to overfit.

33. What does verbose=1 mean?
It tells Keras to print the live animated progress bar while training. Setting it to 0 makes it train silently in the background.

Part 5: Parallel Computing (OpenMP/GPUs) & Complexities
(Since your problem statement asked about Parallel & OpenMP, here is how you connect it!)


34. Does this Python program use OpenMP?
Not explicitly in our code. However, under the hood, the NumPy and TensorFlow/Keras libraries are written in C/C++ and heavily utilize OpenMP and threading to parallelize matrix multiplications on the CPU.

35. How do Neural Networks utilize Parallel Computing?
A neural network is fundamentally just massive Matrix Multiplication (multiplying inputs by weights). GPUs have thousands of cores that can perform these matrix multiplications simultaneously in parallel.

36. What is the difference between CPU (OpenMP) and GPU (CUDA) for Deep Learning?
A CPU (via OpenMP) might have 16 cores to process 16 neurons at once. A GPU (via CUDA) has 5,000+ cores, allowing it to process entire layers of the neural network simultaneously, speeding up training by 100x.

37. What is the Time Complexity (TC) of predicting a new house price?
$O(N \times M)$, where $N$ and $M$ are the number of neurons in adjacent layers. It simply requires a few fast multiplications and additions. Predicting is nearly instantaneous ($O(1)$ practically).

38. What is the Space Complexity (SC) of a trained Neural Network?
$O(W)$, where $W$ is the total number of weights (connections) in the network. Once trained, the model only needs to store these weight values in RAM.

39. Traditional Linear Regression vs. DNN: Which is better?
•	Traditional: Faster to train, mathematically transparent, good for simple data.
•	DNN: Slower to train, acts as a "black box," but significantly more accurate for complex data with non-linear relationships.

Part 6: External Examiner "Trick" Questions


40. Examiner: "Is a DNN with 0 hidden layers just standard Linear Regression?"
Answer: "Yes! If you have inputs connected directly to a single output neuron with linear activation and no hidden layers, the neural network calculates $y = (w_1x_1 + w_2x_2) + b$. That is the exact mathematical equation for traditional Linear Regression."

41. Examiner: "What happens if I forget to scale new_data before using model.predict()?"
Answer: "The model will output a completely wrong, garbage number. The model's weights were trained to understand numbers between -1 and 1. If you feed it a raw RM value of 10, the network will multiply it by the scaled weights and produce an astronomical error."

42. Examiner: "Can I use Cross-Entropy Loss for this problem instead of MSE?"
Answer: "No. Cross-Entropy is strictly used for Classification problems (like predicting probabilities). Mean Squared Error (MSE) is required for continuous Regression problems."

43. Examiner: "If the MAE is 2.5, what does that mean?"
Answer: "Because the Boston dataset stores prices in $1000s, an MAE of 2.5 means the model's predictions are off by $2,500 on average."

44. Examiner: "Why not just use 100,000 neurons to get perfect 100% accuracy?"
Answer: "Using too many neurons causes severe Overfitting. The network will perfectly memorize the exact prices of the training houses but will fail completely when predicting new houses. It also exponentially increases training time."

45. Examiner: "Why use np.array([[0.1, 10.0, 5.0]]) with double brackets?"
Answer: "Because Keras models expect a 2D matrix (batch size, number of features). The outer brackets represent the batch (1 house), and the inner brackets represent the features of that house."

46. Examiner: "What is a 'Loss Function' simply?"
Answer: "It is the mathematical penalty the network receives for guessing wrong. The goal of training is to make the Loss as close to zero as possible."

47. Examiner: "What happens if you run the model.fit() block twice in a row without restarting the program?"
Answer: "The model will continue training from where it left off! So if you ran 100 epochs, and run it again, it is essentially training for 200 epochs."

48. Examiner: "Can you use this network to predict Dog vs. Cat?"
Answer: "Not as it is. We would have to change the final layer's activation from linear to sigmoid, and change the loss function from mse to binary_crossentropy."

49. Examiner: "What is a Hyperparameter?"
Answer: "Settings configured by the programmer before training starts (e.g., number of hidden layers, number of epochs, learning rate). The network cannot learn these on its own."

50. Examiner: "Summarize why Deep Learning is so powerful for this assignment."
Answer: "While traditional linear regression can only draw a straight line, a DNN with ReLU activation can map complex, curvy, non-linear relationships—for instance, acknowledging that going from 1 room to 2 rooms adds some value, but going from 5 rooms to 6 rooms adds massive value."
